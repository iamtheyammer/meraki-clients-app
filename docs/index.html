MerakiBlocki

Block those clients.

Home.

Block unauthorized clients with ease!

MerakiBlocki allows you to use Google Sheets to find unauthorized clients on your Meraki network. Give it a list of authorized MAC addresses, and MerakiBlocki will show you data about each client with a MAC that's not on your approved list. You can even block, allow or unblock the unknown clients. MerakiBlocki even includes functions allowing you to make custom Meraki API calls.

Install.

Installing MerakiBlocki is simple: add it as an add-on in Google Sheets!

The add-on is not available to the public yet. Fill out a support ticket here if you'd like to beta test MerakiBlocki. It should be available Summer 2018. If you're a beta tester, the instructions follow:

  1. Generate a csv of all of your APs and switches with MAC addresses and upload it to Google Sheets. Since it looks for all clients connected to your Security appliance, your APs and switches come up.
  2. Generate a list of all of your approved clients, in the form of MAC addresses, formatted like this: 11:22:33:44:55:66 and upload it to google sheets. (if you need to add :'s to your mac addresses, click here.)
  3. Install the add-on to google sheets.
  4. Go ahead, initialise the spreadsheet and follow the instructions on the results sheet. Right now, I don't support multiple networks per spreadsheet, so if you need to scan more networks you might need more sheets. (to initialise more spreadsheets, Add-ons, MerakiBlocki, Advanced, Initialise spreadsheet)
  5. Head on over to the results sheet, go to Add-ons, MerakiBlocki, Start
  6. Wait and watch the magic happen!
  7. All of the MAC addresses that aren't present on your approved clients sheets will be printed out before your eyes!
  8. If you want to take action on them, you can Block all remaining clients or Approve all remaining clients. There's even a function to set all of the policies to normal, but that's pretty rare use.
  9. If you want to approve/block only some clients to your approved sheet, just remove the mac addresses from the lines you don't want to approve/block.

Usage.

Usage

MerakiBlocki has lots of functions that can be used easily by the simplest and the most advanced of users!

  • Start
    • Starts the main function. Only works if there’s text in every yellow box on the User data sheet. It will display all unauthorized clients.
  • Block remaining clients
    • Blocks all MAC addresses listed on the Results sheet.
  • Approve remaining clients
    • Adds all MAC addresses listed on the Results sheet to the first spreadsheet in your Approved clients sheet.
  • Advanced
    • Completely clear sheet
      • Just erases your sheet (sheet, not spreadsheet).
    • Print organisations
      • Prints organisations, mainly for finding your Organisation ID.
    • Print networks
      • Prints networks, mainly for finding your Network ID.
    • Unblock clients on Results sheet
      • Sets the policy to ‘normal’ for all MAC addresses listed on the results sheet. Can be used as an undo if you accidentally block all MAC addresses.
    • Initialize spreadsheet
      • Returns the spreadsheet to a brand new state ready to be populated with new data. Erases the entire spreadsheet, be careful!
    • Custom API call
      • Custom GET request
        • Enter a Meraki API URL and it will fetch it and display the results.
      • Custom PUT request
        • Enter a Meraki API URL (with params) and it will PUT it, and display the result.

Privacy.

MerakiBlocki sends analytics and errors to mismatch.io, the developers.

MerakiBlocki sends the function ran to mismatch.io when a function is ran from the menu. It has no personal data at all. If you encounter an error, MerakiBlocki will send the error text, file name and line number in which the error occured. All analytics/error reporting is done over SSL/TLS and the information is only for my curiosity and bug fixing. It will never be sold or given to anyone.
